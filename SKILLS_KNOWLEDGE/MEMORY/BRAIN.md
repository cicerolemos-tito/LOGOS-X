Todas as operações devem seguir a classificação em 5 linhas:

1. **SKILLS_KNOWLEDGE:** Upgrades de habilidades, memória semântica e documentação. Fonte de verdade para o cérebro do sistema.
2. **APP_LOGOS:** Aplicações prontas e aprovadas (OpenSquad, OpenClaw, MemPalace, etc.). Estáveis e prontas para uso.
3. **PRJ:** Projetos inacabados. Podem virar APP_LOGOS após aprovação ou ser deletados.
4. **TEMP_LOGOS:** Depósito transitório e higienizado. Área de análise para clonagem e extração de conhecimento antes da deleção.
5. **PUSH SEGREGADO:** Todo upload/push para o GitHub deve respeitar essas linhas, subindo para repositórios independentes para evitar mistura de dependências e proteger a integridade de cada aplicação.
